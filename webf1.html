Second week - Web Security

I learned what are the main social engineering techniques:

  • Phishing, impersonation
  • Physical access (tailgating)
  • Shoulder surfing
  • Dumpster diving, stealing important information
  • Fake software, trojans
  • Quid pro quo, baiting, pretexting

Social engineering cannot be blocked by technology alone, so we have to:

  • Decide what information being is published about us or our family online
  • Find out what information about us and our family is available to the public
  • Remove metadata from pictures before you post them online
  • Protect our phone number
  • Think before we click
  • Check privacy settings regularly and change them from their default settings
  • Keep passwords safe by keeping them out of sight,changing them regularly and don’t let anyone else use them
  • Do not give social media apps access to our phone or email address
  • Hand over personal information wisely
  • Make a plan for what to do if we lose our device
  • Don’t make our device easy for others to access

Fourth week - Academic Writing

How to write better (in academic english):

  • Phrasal verbs increase the word count so they can often be replaced by better words with the same or more appropriate meaning
  • Choose appropriate adjectives/expressions
  • Choose formal words and phrases
  • Replace the contractions with full words
  • Avoid personal and subjective writing
  • Rewrite bold statements so that they can be more cautious
  • Rewrite sentences so they are accurate, cautious and formal

Thirteenth week - Addressing the Web

I learned that:

  • Every computer on the internet has a unique(ish) IP address.Global internet hostnames are translated to IP addresses using DNS
  • Hostnames can be local and global
  • The scheme part of the URL indicates the format in which the request should be sent. Common supported ones are:
    • ftp://
    • http://
    • https://
    • file://
    • mailto:
  • Old ones we no longer see:
    • aol://
    • gopher://
  • DNS (Domain Name Service) is a global record of domains to IP addresses.Until recently, domains were allocated under countries,but some names are less restricted.
  • Computers run network services on ports.Port numbers under 1024 are reserved for specific purposes
  • Usernames and passwords can also be provided as part of an URL
  • Query strings can also be passed via a URL, allowing server-side scripts to interpret them as variables,like:
  • HyperText Transfer Protocol is the underlying protocol for the World Wide Web
  • When a URL is entered into the address bar of a user, you are sending a HTTP GET request to the server.
  • The fragments are also a part of the URL.
  • There are two types of URLs:
    • Absolute,which must contain the scheme (e.g. http://)
    • Relative,which specifies a URL relative to the page.
  • On Windows systems, file names are not case sensitive,but on Linux systems, file names are.

I did some additional research on the web standards:

  • Web standards are rules and guidelines established by the World Wide Web Consortium (W3C) developed to promote consistency in the design code which makes up a web page.
  • The advantages in adhering to these standards are many:
    • Web pages will display in a wide variety of browsers and computers, including new technology.
    • W3C Standards promote the use of "Cascading Style Sheets" (CSS) or design code which is attached to the web page rather than embedded in the page.
    • Design features such as colors and fonts can be easily changed by just modifying one style sheet instead of editing every individual page in a web site, reducing the costs to modify your site.
    • Search Engines are able to access and index pages designed to web standards with greater efficiency.

508 section/rule

  • In 1998, Congress amended the Rehabilitation Act to require Federal agencies to make their electronic and information technology accessible to people with disabilities. Inaccessible technology interferes with an individual's ability to obtain and use information quickly and easily. Section 508 was enacted to eliminate barriers in information technology, to make available new opportunities for people with disabilities, and to encourage development of technologies that will help achieve these goals.

I started by reasearching what JavaScript is:

  • JavaScript is a simple and easy-to-learn programming language as compared to other languages such as C++, Ruby, and Python.
  • It is a high-level, interpreted language that can easily be embedded with languages like HTML.
  • Initially, JavaScript was known as LiveScript, but later Netscape changed its name to JavaScript.

Some amazing features of JavaScript are as follows:

  • Easily embedded with languages like HTML, CSS, and Java.
  • Also works as a client-side scripting language, which helps in creating dynamic web pages.
  • JavaScript is an open source and cross-platform language and therefore mostly used for creating network-centric applications. Nowadays it has become an inseparable part of the core technologies of the internet.
  • JavaScript is used in web technologies to make web pages interactive. Along with HTML and CSS, JavaScript is widely used in web servers and databases. Hence its wide popularity among these kinds of websites.
